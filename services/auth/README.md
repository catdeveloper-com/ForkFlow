# Сервис аутентификации ForkFlow

## Текущий статус

Этот каталог резервирует границу для будущего сервиса аутентификации ForkFlow.
Foundation PR намеренно не содержит исходного кода, runtime-зависимостей,
маршрутов, моделей базы данных, миграций или бизнес-логики.

## Планируемая ответственность

Сервис аутентификации будет отвечать за идентификацию и аутентификацию:
регистрацию пользователей, проверку учётных данных, жизненный цикл токенов и
сессий, а также роли авторизации. Другие сервисы не должны напрямую обращаться
к его базе данных.

## Планируемые внутренние границы

При реализации сервис будет следовать направлению зависимостей Clean
Architecture, принятому в репозитории:

```text
presentation -> application -> domain
infrastructure -> application/domain through ports
```

- `domain` будет содержать независимые от фреймворков сущности, value objects
  и доменные ошибки.
- `application` будет содержать use cases, DTO и порты.
- `infrastructure` будет реализовывать хранение данных и внешние интеграции.
- `presentation` будет содержать HTTP-схемы и тонкие обработчики маршрутов.

## Отложенная работа

Первый PR с реализацией Auth должен добавить только исполняемый каркас сервиса,
конфигурацию, healthcheck и соответствующие тесты. Сценарии аутентификации и
хранение данных должны появляться отдельными сфокусированными изменениями.
